# Granular Design System

## Reference Direction

The site takes inspiration from GitBook's current product-marketing language: a slim announcement bar, calm rounded navigation, large editorial hero copy, product interface mockups, bento-style feature panels, proof sections, and a final conversion call-to-action.

This is adapted for Granular rather than copied directly. The system uses Granular's logo, business-plan content, privacy-first positioning, and receipt intelligence product story.

## Brand Feel

- Clear, calm, and high-trust
- Documentation-like, but still polished as a product site
- Privacy-first and financially practical
- Structured around knowledge, control, and staged rollout

## Core Tokens

- Background: warm paper `#F6F4ED`
- Surface: near-white `#FEFEFC`
- Ink: dark charcoal `#151A1D`
- Muted text: `#69746E`
- Trust green: `#1F6F55`
- Deep green: `#173F35`
- Amber accent: `#F3C35A`
- Coral accent: `#E06A4F`
- Border: `#DED9C9`
- Radius: 8px for controls, 24-36px for large product panels

## Components

- Announcement bar
- Sticky translucent header
- Rounded nav links
- Solid and ghost action buttons
- Hero release pill
- Browser-style product mockup
- Operating-layer marquee row
- Bento system cards
- Sticky document navigation
- Long-form document card
- Final CTA with product flow illustration

## Responsive Rules

- Desktop keeps the GitBook-like nav, actions, and side document navigation.
- Tablet stacks the hero and compresses cards into one column.
- Mobile swaps the desktop nav for a hamburger menu, locks scroll while open, and keeps all document content in single-column cards.
